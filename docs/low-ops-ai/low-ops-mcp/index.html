Low-Ops MCP

Low-Ops MCP is an MCP server that exposes Low-Ops platform operations to MCP-compatible clients. MCP is an open standard for connecting assistants to tools and data; see the Model Context Protocol documentation for background.

What you can do​

With the Low-Ops MCP server enabled in your client, an assistant can use tools to work with applications and environments you are allowed to access in Low-Ops.

Tools​

ToolPurpose
applications_listList applications you can access (optional search and pagination).
applications_createStart creating an application from a template (name, templateId; optional description). Can take up to about five minutes.
templates_listList available templates (optional search and pagination).
environments_listList environments for an application (appName required).
environments_createStart creating an environment (appId, name). Can take up to about three minutes.
environments_deployStart a deployment (environmentId, packageId). Requires environment lifecycle to be created; can take up to about five minutes.
environments_startStart an environment (environmentId).
environments_stopStop an environment (environmentId).
environments_restartRestart an environment (environmentId).
packages_listList deployable packages for an application (appId required; optional search and pagination).
source_repository_urlGet the source repository URL for an application (appName required).

Prompts​

The server also advertises prompts (reusable task starters) such as listing applications, retrieving environments, planning a deployment, and deploying an environment. Exact behavior depends on your MCP client.

Authentication and access​

The MCP server uses your Low-Ops identity. You must be able to sign in to Low-Ops with the same account your client uses for MCP (how credentials are supplied is configured in your MCP host). Access is limited to what your user can do in the platform, as described under Authorization.

Client configuration​

Configuration is specific to each MCP host (Cursor, Claude Desktop, and others). Here we will provide instructions for Claude Desktop and Cursor because they are the most popular MCP hosts.

Claude Desktop​

To use the Low-Ops MCP server in Claude Desktop, you need to add Custom connector to your Claude Desktop configuration:

Go the the Settings > Connectors and Add Custom Connector.

Fill in the connector details: Name and URL and press Add. Name can be anything you want, URL should be your Low-Ops Instance URL.

To start using the connector (MCP) you have to Connect (Authonticate) to the Low-Ops. So press Connect , you will be redirected to the Low-Ops login page. Sign in with your Low-Ops credentials.

After successful authentication, you will be redirected to the Claude Desktop and the connector will be connected. You can now use the connector to interact with your Low-Ops MCP.

Cursor​

To use the Low-Ops MCP server in Cursor, you need to add mcp.json file to your Cursor configuration.

Go the the Cursor Settings > Tools & MCP's press to New MCP Server.

Add the Low-Ops MCP Server URL to the mcp.json file.

Go back to the Cursor Settings > Tools & MCP's page and press Connect to connect to the Low-Ops MCP.

After successful authentication, you will be redirected to the Cursor and the MCP server will be connected. You can now interact with your Low-Ops MCP.
